# Competitors and Substitutes

## Direct Competitors

| 来源 | 原始文件可内置 | 署名 | 适合 MoonCut 的位置 |
|---|---:|---:|---|
| HoliznaCC0 / FMA | 是，逐曲 CC0 | 不要求 | 首版 Core Pack |
| Incompetech | 是，CC BY 4.0 | 必须 | 第二批开放曲库，需自动 credits |
| Audionautix | 条款有额外限制，公开裸文件应谨慎 | 必须 | 受控构建或先书面确认 |
| Mixkit | 不宜聚合/库存化再分发 | 不要求 | 最终成片按项目使用，不进 Core Pack |
| Pixabay | 禁止 standalone 分发 | 不要求 | 用户导入或按项目下载 |
| Scott Buckley | 官方要求与媒体同步、禁单独再分发 | 必须 | 具体成片配乐，不进 Core Pack |
| 抖音 / 剪映平台音乐 | 否 | 依素材页面 | 热度与风格参考；用户发布时在平台内加歌 |
| HIFIVE 音加加 | 需企业书面合同 | 依合同 | 首选国内 API/SDK 与短视频授权候选 |
| Vfine Music | 需企业书面合同 | 依合同 | 第二家询价与曲库补充候选 |
| TME 音乐云 / 曲易买 | 需逐项/企业授权 | 依合同 | 华语主流与定制曲库，高质量但可能更贵 |
| 100Audio | 默认按项目，不宜直接视为 SaaS 曲库 | 依授权等级 | 单条成片或少量采购，须另谈多用户输出 |

## Indirect Substitutes

- 用户自带本地音乐：权利责任清楚时最灵活，但元数据与响度不统一。
- 付费订阅音乐库：曲目质量和检索通常更好，但 SaaS 转授权、席位和输出平台范围需要单独合同。
- 定制/AI 生成音乐：可获得更贴内容的结构，但要核验模型条款、训练来源、独占性和 Content ID 政策。
- 完全不用 BGM：对高信息密度或严肃口播有时反而更清晰，应作为自动策略的合法选择。
- 平台加歌：MoonCut 可输出无 BGM 成片，并给出推荐曲名、搜索关键词和建议起播秒数，让用户在抖音/小红书/B站内使用平台许可。

## Manual Workflows

传统流程是剪辑师先理解内容情绪，再去平台试听、下载、记录许可、手动调音量和关键帧。自动化应把这一过程拆成内容意图分类、候选排序、语音频段评估、按 ASR 做 duck、授权记录和导出说明。

## Positioning Notes

MoonCut 的差异点不应只是“有很多歌”，而应是两层产品：一层是可以直接导出的国内审美授权曲库；另一层是平台热门曲推荐（只给曲名/版本/起播点，不打包音频）。两层都应按内容赛道匹配，并自动保障人声可懂度。
