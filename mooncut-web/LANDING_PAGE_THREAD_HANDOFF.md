### 四、Landing Page 的页面架构

新增独立组件，建议：

```text
src/components/LandingPage.vue
```

必要时可拆少量 Landing 专用子组件，但不要为了形式过度拆分。Landing Page 至少包含以下区域：

#### 1. 顶部导航

- MoonCut 品牌标识。
- 页面锚点：“功能”“工作流”“隐私与本地演示”。
- 主要按钮：“开始创作”。
- Landing Page 不显示工作区的主题切换器，也不显示移动端工作区底栏。
- 顶部导航在桌面端轻微悬浮/半透明；移动端简化为品牌标识与一个 CTA，避免拥挤。

#### 2. Hero 首屏

建议文案：

```text
AI 口播创作工作台

从一个想法，
到一条能发的口播。

一起聊出脚本、对着提词器录下来，
再把停顿、重复和字幕交给 MoonCut。
```

两个明确入口：

- 主按钮“开始创作” → 进入 `record` 页面，并优先呈现口播构思/脚本创作入口。
- 次按钮“直接剪视频” → 进入 `edit` 页面。

首屏右侧或下方制作一个真实、精致的产品流程演示面板，表达：

```text
想法 → 脚本 → 提词录制 → 智能剪辑 → 成片
```

面板可使用 HTML/CSS 构建动态脚本行、音频波形、时间线、字幕片段和处理状态，不依赖远程图片，不制作虚假的不可交互输入框。

#### 3. 价值信息条

用紧凑的信息条呈现：

- 一起聊出内容
- 提词录制
- 精简停顿与重复
- 字幕与节奏包装
- 本地演示

“本地演示”必须清晰可见，避免把 Demo 描述成已经上线的云端 AI 服务。

#### 4. 三项核心能力

三张能力模块，不做普通同质化小卡片堆叠，要用大小、排版和视觉结构形成层级：

1. **聊想法，长成一篇能说的话**  
   从主题、观点和语气出发，整理成自然口语脚本。
2. **看着稿，也像在和人说话**  
   提词器、镜像、倒计时和摄像头录制形成一条连贯路径。
3. **只剪影响节奏的部分**  
   识别停顿、重复与口头语，并完成字幕和节奏包装。

每个模块应使用与现有真实功能一致的产品片段或 CSS 微型界面做展示，不要使用无意义的图库照片。

#### 5. 完整工作流

用一条清晰的时间线说明：

```text
01 说出想法
02 整理成稿
03 提词录制
04 自动精简
05 检查并导出
```

强调它是一条连续创作路径，而不是五个互不相关的功能。

#### 6. 产品实景展示

至少安排两段错落展示：

- 口播助手 + 提词录制。
- 上传素材 + 文本剪辑/字幕时间线。

优先复用现有工作区的视觉语言或以纯 CSS 构造高保真预览。不要复制完整业务组件导致状态耦合，也不要从外网临时引用图片。

#### 7. 陪伴感与宠物

现有 `PetCompanion.vue` 是产品体验的一部分。Landing Page 可以让宠物在首屏产品演示附近或工作流段落中出现一次，承担“陪你完成创作”的情绪锚点：

- 宠物不能遮挡标题、按钮和重要内容。
- 保留点击/抚摸后的开心反馈。
- 移动端应缩小或移动到安全区。
- 不要让页面同时出现两个宠物实例。
- 如果现有全局宠物直接复用更稳妥，就只调整 Landing 下的位置和层级，不重写动画系统。

#### 8. 隐私与本地演示

用一个简洁区域说明当前 Demo 的边界：

- 素材在浏览器本地演示流程中处理。
- 用户可以先体验脚本、录制和剪辑路径。
- 不夸大真实模型能力、服务端处理或数据安全承诺。

#### 9. 最终 CTA 与页脚

结尾使用一句强收束文案，例如：

```text
下一条口播，从一句还没想完整的话开始。
```

按钮：

- “开始创作” → `record`
- “上传视频” → `edit`

页脚只保留品牌、Demo 状态和必要说明，不添加虚构公司信息、社交媒体或法律链接。

### 六、唯一的 Landing 视觉方向

Landing Page 使用一套独立、固定的“精密创作台”科技视觉，不随 `light / dark / memphis` 切换。

建议设计令牌：

```css
--landing-bg: #070A12;
--landing-surface: #0E1422;
--landing-surface-2: #121C2D;
--landing-text: #F5F8FF;
--landing-muted: #9AA8BC;
--landing-cyan: #62E7FF;
--landing-blue: #6D8CFF;
--landing-lime: #B8F35A;
--landing-line: rgba(255,255,255,.10);
```

视觉要求：

- 深海军蓝/近黑底色，清晰的白色排版，电光青与少量荧光绿做功能状态色。
- 使用克制的网格、细线、时间轴刻度、波形和状态光点形成科技感。
- 可以有轻微径向光晕，但禁止整页套用廉价大面积渐变。
- 字体优先沿用项目现有字体栈，不为了 Landing 引入沉重字体依赖。
- 标题要有明确字号对比；正文行长控制在易读范围。
- 面板边框要精细，不使用无意义玻璃拟态堆叠。
- 不把 Landing 做成 Memphis 风，也不需要为 Landing 另外做浅色版。
- 避免大量等宽小卡片、无差别圆角矩形、模板化 SaaS 首页感。

Landing 样式必须收口在独立根类（例如 `.landing-shell`）下，避免污染三个工作区主题。现有主题 CSS 只在工作区生效。

### 七、动效计划

动效服务于“创作流正在运转”，不能抢夺阅读注意力：

- 首屏标题与 CTA：进入时轻微上移 + 淡入，约 `360–480ms`。
- 产品演示：脚本行、波形游标、时间线处理点按顺序循环，单轮 `1.8–2.6s`。
- 工作流：滚动进入时逐项点亮，最多一次，不持续闪烁。
- 按钮按下反馈：约 `100–140ms`。
- 宠物继续使用现有帧动画和互动反馈。
- 所有非必要动效必须遵守 `prefers-reduced-motion: reduce`，在该模式下取消循环、位移和滚动揭示。
- 不引入重型动画依赖；优先 CSS transition/keyframes 和少量 Vue 状态。

### 八、页面状态与导航实现

保持当前轻量状态架构，不要为了三个内部页面强行引入 Vue Router，除非你先验证确实有必要。

建议实现：

1. 将 `WorkspacePage` 扩展为：

   ```ts
   export type WorkspacePage = 'landing' | 'edit' | 'record'
   ```

2. `App.vue` 每次完整加载默认 `activePage = 'landing'`。用户明确要求“一进来就是 Landing Page”，因此不要再从 `mooncut:page` 把首次页面恢复成 `edit` 或 `record`。
3. 主题偏好、演示素材和已有工作区状态可以继续保存；只是完整刷新后入口回到 Landing。
4. 从 Landing 的 CTA 进入对应工作区。
5. 工作区桌面导航的品牌标识应可以返回 Landing；移动端增加一个清晰但不拥挤的“首页/返回首页”入口。
6. Landing 不显示工作区底部导航；进入 `edit`/`record` 后继续使用现有导航和主题切换。
7. 优先继续用 `v-show` 或其他不会意外销毁现有工作区状态的方式切换。不得因为返回 Landing 就丢失已加载视频、脚本、剪辑结果或正常录制状态。
8. 如果正在录制、倒计时或处理中，返回 Landing 的行为要安全：至少避免无提示销毁媒体流或任务。复用现有停止/清理机制，不要新增复杂弹窗体系。

建议 CTA 映射：

| Landing 操作 | 目标页面 | 预期状态 |
|---|---|---|
| 开始创作 / 从想法开始 | `record` | 显示口播构思或脚本创作入口 |
| 直接剪视频 / 上传视频 | `edit` | 显示上传空状态 |
| 点击品牌标识 / 返回首页 | `landing` | 保留现有工作区数据，显示 Landing |

### 九、响应式、可访问性与内容要求

- 重点验收 `1440×900`、`390×844`、`320×568`。
- 任何宽度不得横向溢出。
- 移动端 CTA 和可点击控件至少 `44×44px`。

- 不要把核心文案放进 Canvas 或图片。
