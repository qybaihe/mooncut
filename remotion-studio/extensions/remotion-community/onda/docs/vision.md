# Vision

## What Onda is

Onda is an open-source library of **production-ready motion graphics components for [Remotion](https://remotion.dev)**. People run `npx ondajs add <name>` and the component **source is copied into their own project**, theirs to own and edit. Onda is **not** an npm dependency imported as a black box.

**Positioning:** the curated, best-in-class, code-first motion graphics library for developers and AI coding agents.

**Distribution model.** Our own thin CLI is the primary install path; the docs site doubles as a registry with one-click copy-paste. We adopt the [shadcn](https://ui.shadcn.com) registry JSON format on disk so the shadcn CLI works against our registry as a bonus — but we don't require users to set it up. Rationale: [techspecs/002-distribution-model/design.md](techspecs/002-distribution-model/design.md).

## The differentiator — signature motion identity

Onda's edge is NOT exotic or never-seen components. It is a *recognizable way everything moves* — a consistent set of motion fingerprints applied across ordinary components, so someone seeing an Onda animation knows it's Onda by feel alone, before any logo appears (the way Apple, Linear, or Stripe motion is recognizable by feel).

Uniqueness lives in **how it moves**, not **what it is**. Never chase weird components for novelty — that reads as gimmicky. Chase a flawless, coherent, unmistakable motion signature applied to a clean set of components.

## What we are NOT building

- A render farm, hosted editor, auth, billing, or backend. v1 is **library + static docs site only**.
- A vast catalog. We win on **taste and coherence**, not volume. 40 perfect components beat 4,000 mediocre ones.
- Clones of existing commercial templates, studio reels, or marketplace assets. Every component is **original work** built from motion *principles*, never copied from a specific existing graphic.
